# 即梦 Seedance 2.0 视频生成指南

平台地址：jimeng.jianying.com（国内）/ dreamina.capcut.com（国际版 Dreamina）
模型：Seedance 2.0（字节跳动，2026年2月发布）

---

## 为什么用 Seedance 2.0

与 Sora / Kling / Runway 等工具相比，Seedance 2.0 的核心优势是**多模态全能参考**——你不再只能靠文字描述，而是直接把参考素材扔给它，用 @ 指向具体素材，它照着理解意图、自动规划分镜和运镜。

**与其他工具最大的不同：**

| 能力 | Seedance 2.0 | 其他工具 |
|------|-------------|---------|
| 输入方式 | 图片 + 视频 + 音频 + 文字，四路并联 | 通常只有文字 + 首帧 |
| 运镜控制 | 上传参考视频，直接复刻镜头语言 | 只能靠提示词描述术语 |
| 口播同步 | 上传音频 + 人像，原生唇形同步生成 | 需后期对口型 |
| 多镜头叙事 | 一条提示词可包含分镜切换 | 通常只能生成单镜头 |
| 中文理解 | 原生支持，效果极好 | 大多数为英文优先 |
| 负面提示词 | **不支持**（别写"不要什么"） | 大多支持 |

## 生成参数

| 参数 | 选项 | 建议 |
|------|------|------|
| 时长 | 4–15 秒 | 调试期用 5 秒，确认效果后再用 10–15 秒 |
| 分辨率 | 480p / 720p / 1080p / 2K | 480p 调试，1080p 正式出品，2K 最终版 |
| 画面比例 | 16:9 / 9:16 / 1:1 | 先定好平台，不要生成完再裁剪 |
| 音频 | 默认开启 | 要自己配乐配音就关掉 |

> ⚠️ 每次生成最多 15 秒。更长视频用**延长**功能接续生成（见后文）。

## 两种使用模式

**首尾帧模式：** 上传起始帧图片 + 结束帧图片，模型生成中间过渡动画。适合需要精确控制画面首末状态的场景。

**全能参考模式（主推）：** 同时上传图片、视频、音频多种素材，用 @ 指定每个素材的用途，写提示词描述想要的效果。Seedance 2.0 最强能力在这里。

---

## @ 引用语法——最核心的操作

上传素材后，必须在提示词里用 `@素材名` 明确告诉模型每个素材怎么用。模型不会自己猜，不说就乱用。

**触发方式：** 输入框里打 `@`，会弹出已上传素材列表，点击选择。

**@ 用途类型：**

```
@图片1 作为角色形象          → 指定人物长相/服装/风格
@图片2 作为场景背景          → 指定环境风格
@视频1 参考运镜手法          → 复刻镜头语言和运动轨迹
@视频1 参考动作节奏          → 模仿动作/打斗/舞蹈节奏
@音频1 同步口型              → 角色口型与该段语音对齐
@音频1 作为背景音乐          → 跟音乐节拍生成画面节奏
```

**文件上限：** 总计 12 个文件（图片最多 9 张 + 视频最多 3 段（≤15s） + 音频最多 3 段（≤15s））

> 优先上传对画面和节奏影响最大的素材，图 3–5 张 + 视频 1–2 段 + 音频 1 段是常用配置。

## 提示词公式

Seedance 写中文即可，不需要堆砌英文关键词。公式：

```
[主体] + [动作/状态] + [场景/环境] + [光影] + [镜头语言] + [风格/质感] + [约束/画质]
```

**完整示例：**
```
一位穿白色亚麻连衣裙的年轻女生，长发微卷，站在春日午后的日式庭院木廊上，
樱花花瓣缓缓飘落在肩头。近景，缓慢推镜，暖光从侧面洒入，柔光散射，
日系清新暖色调，画面稳定无抖动，4K超高清，面部清晰不变形，电影质感。
```

### 各维度写法示例

**主体描述（写具体，不写泛化形容词）：**
```
❌ 一个漂亮女孩
✅ 一位穿黑色皮衣的年轻女性，短发，东亚面孔，表情冷静
```

**动作描述（动作越慢越稳，越复杂越容易翻车）：**
```
✅ 稳定表现：缓慢行走 / 微笑点头 / 轻轻转身 / 举起咖啡杯
⚠️ 容易翻车：快速跑动 / 剧烈跳跃 / 多人打架 / 大幅舞蹈
```

**镜头语言（支持中文直接描述）：**
```
推镜        → 缓慢推镜 / 快速推进
拉镜        → 慢慢向后拉远
横移        → 镜头平滑向右横移
环绕镜头    → 以人物为中心环绕一圈
俯拍        → 从高处俯视，鸟瞰视角
仰拍        → 从低处仰望
跟拍        → 镜头平稳跟随人物移动
固定镜头    → 镜头静止不动，只有人物动
多镜头切换  → 第一个画面[描述]，第二个画面[描述]，第三个画面[描述]
```

**光影描述（对氛围影响极大）：**
```
黄昏暖光 / 清晨侧光 / 正午硬光 / 阴天漫射光
室内灯光氛围感 / 背光轮廓光 / 窗帘透光丁达尔效应
```

**风格/质感关键词：**
```
电影感 / 日系清新 / 港片质感 / 国产武侠 / 赛博朋克 / 极简现代
胶片颗粒 / 暖色调 / 冷色调 / 高饱和 / 低饱和高对比
```

**约束/画质关键词（重要，防止常见问题）：**
```
画面稳定无抖动         → 防止画面颤抖
面部清晰不变形         → 防止人脸形变
人体结构正常           → 防止肢体扭曲
细节丰富               → 提高整体清晰度
4K超高清 / 2K清晰      → 提升画质要求
电影质感               → 整体质感提升
```

## 多素材组合场景模板

### 场景一：角色一致性短视频

**目标：** 保证多镜头里同一人物外观一致

```
素材：@图片1（人物正面半身照）+ @图片2（场景环境图）

提示词：
@图片1 中的女孩在 @图片2 的咖啡馆里，坐在靠窗位置，
手捧咖啡杯，视线望向窗外，若有所思的表情。
近景，镜头缓慢向右横移，午后暖光从侧面透入，
柔和散射，暖棕色调，画面稳定无抖动，电影感。
```

### 场景二：复刻运镜（最强项）

**目标：** 把参考视频的镜头语言移植到新场景

```
素材：@视频1（参考镜头片段，≤15s）+ @图片1（新场景主体）

提示词：
参考 @视频1 的运镜方式和镜头节奏，
用 @图片1 中的小米 SU7 Ultra 复刻这段视频，
保持相同的镜头运动轨迹，背景换成夜晚城市高架桥，
霓虹灯光倒影在湿润路面，电影感追车镜头。
```

### 场景三：口播视频（原生唇形同步）

**目标：** 角色口型与旁白音频精准同步

```
素材：@图片1（人物正面半身照，嘴巴微闭）+ @音频1（旁白录音）

提示词：
@图片1 作为角色形象，面对镜头说话，
口型与 @音频1 精准同步，表情自然亲切，
背景为简洁的白色书房，镜头稳定不动，
面部清晰自然，不变形。
```

### 场景四：视频延长（接续生成长视频）

**目标：** 把已生成的短视频延长，保持一致性

```
素材：@视频1（已生成的片段）+ @音频1（下一段旁白）

提示词：
将 @视频1 延长 10s，角色继续面对镜头说话，
口型与 @音频1（第二段台词）精准同步，
保持角色外观、背景、光线与前段完全一致，
镜头保持稳定不变。
```

> ⚠️ 生成时长选择"新增部分"的时长。延长 10 秒就选 10 秒，不是选总时长。

### 场景五：多镜头叙事（一条 prompt 含分镜）

**目标：** 一次生成多个有叙事联系的镜头片段

```
第一个画面：从咖啡杯特写开始，热气缓缓升起，镜头缓慢上移。
第二个画面：切换到坐在咖啡馆里捧杯的女孩中景，望向窗外。
第三个画面：窗外街道全景，人来人往，镜头慢慢虚焦。
整体风格：暖色日系，午后光感，电影感，画面稳定。
```

## 长视频工作流（15 秒限制的解法）

Seedance 单次最多生成 15 秒。做更长的视频用**延长（Extend）**逐段接续：

```
第一段：生成 5–10 秒基础镜头
↓
点"延长"→ 填写下一段提示词（描述接续内容）→ 生成时长选"新增秒数"
↓
重复延长，直到达到目标时长
↓
在剪映 / CapCut / Premiere 里拼接（即梦和剪映是同一家，导入更顺滑）
```

**保持连贯性的关键词（加在延长提示词末尾）：**
```
保持角色外观与前段完全一致
动作自然连贯，与前段无缝衔接
镜头风格与前段一致
光线和色调保持前段风格
```

## 常见问题排查

| 问题 | 原因 | 解决 |
|------|------|------|
| 人脸变形/扭曲 | 画质要求不够 | 加 `面部清晰不变形，人体结构正常` |
| 画面颤抖 | 未限定稳定性 | 加 `画面稳定无抖动，镜头平稳` |
| AI 感明显 | 描述太笼统 | 加具体光影 + 风格词，减少抽象形容词 |
| @ 引用没起效 | 提示词里没写 @素材名 | 检查每个素材都有对应的 @ 引用 |
| 动作变形 | 动作幅度太大 | 改用小幅缓慢动作，复杂动作用 @视频 参考 |
| 口型不同步 | 音频太长或人脸角度不佳 | 音频保持 ≤15s，人脸正面，嘴巴自然微闭 |
| 生成等太久 | 高峰时段（20:00–23:00） | 换低峰期，或用 Seedance 2.0 Fast 模型 |

## 与 Kling / Sora / Runway 的提示词差异

| 工具 | 提示词风格 | 特点 |
|------|-----------|------|
| **Seedance 2.0** | 中文自然语言 + @多模态引用 | 描述故事，AI 自动规划分镜运镜 |
| **Kling** | 中英文均可，结构化描述 | 需写清楚镜头类型和运动方式 |
| **Sora** | 英文为主，细节描述 | 对场景细节和物理描述响应好 |
| **Runway** | 英文，支持 Motion Brush | 结合手动控制区域运动 |

**结论：** 给即梦的提示词不需要堆砌专业摄影术语——直接用中文说故事，它会规划分镜；运镜复杂就上传参考视频，比写"环绕镜头 + 跟焦 + 景深变化"更精准。
